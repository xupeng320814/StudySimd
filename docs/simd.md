AVX512的支持情况并不足够，在这篇文章中并不将其纳入考量。

认为2021年后，应认为一定使用 SSE,包括SSE4.1
而AVX的使用需要考虑平台兼容性等。

SIMD的理想工作模式：
        源数据加载到寄存器
        寄存器中完成尽可能多的操作
        将结果存储会内存或者通用寄存器

SSE
存在下面几种类型：
                                16 bytes                32 bytes
32-bit float            __m128                  __m256
64-bit float            __m128d                __m256d
Integers                __m128i                  __m256i
表示寄存器的大小以及其中元素的类型以及大小。

需要注意的是，底层只有16个寄存器可用。（64位程序）
如果局部变量过多或者依赖严重，会将部分变量存储到栈内存各种，这可能会破坏性能。
需要注意的是，并不存在独立的16字节寄存器，这个寄存器其实就是32位寄存器的低半部分。
【但是C++用不同的类型表示，见上面的描述】

可以通过指令进行类型间的转换
比如__m128d 可以使用 _mm_castpd_ps 转换成__m128
类似这种转换函数，是没有成本的
128 可以转换成256，只是高半部分是未定义的而已。
